-Select Next button

The Next button process directs the user to the User Access tab.

The User Access tab determines which cases or team project portal the user will have access to.  This tab contains the following two subtabs where additional case and team level security access rights can be applied:
 

-My Cases - this subtab will only show if the Role Type is Case Manager of the selected User Role

 

-This section will list all the clients and associated cases that are added under the Client Management and Case Management submodules

-In order for a user to have access to a case portal under the My Cases tab, one or more options must be checked.

-A user can gain access to a case in the following ways:

 

1.   Create a new client and new case.  If a user creates the client/case themselves, the system will automatically grant the right to the user to access

2.   Select the Automatically Inherent New Cases option - by selecting this option, any new client/case that is added, the system will automatically grant the right to the user to access

3.   If the Automatically Inherent New Cases option is not selected, anyone that has rights to the Manage User section can either assign a client/case to themselves or assign rights to other users

-My Teams - this subtab will only show if the Role Type is Team Member of the selected User Role
 

-This section will list all the teams that are added under the Workflow Management -> Operation Teams submodules

-In order for a user to have access to a team portal under the My Teams tab, one or more options must be checked.

-A user can gain access to a team in the following ways:
 

1.   Create a new team.  If a user creates the team themselves, the system will automatically grant the right to the user to access

2.   Select the Automatically Inherent New Teams option - by selecting this option, any new team that is added, the system will automatically grant the right to the user to access

3.   If the Automatically Inherent New Teams option is not selected, anyone that has rights to the Manage User section can either assign a team to themselves or assign rights to other users
