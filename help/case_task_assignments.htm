Case Task Assignments

 Top  Previous  Next

The Case Task Assignments section allows users to assign or transition tasks to users in Track Project submodule.  This process must be performed first in order to begin working on tasks status operations.

 

 

-Select a Task row

 

-Select the Assign/Transition Task node in left section

 

 

There are two groups of users that are displayed to select from and assign/transition tasks to:

 

1.        My Cases module, the first group of users will be only the Role Type of Case Managers and the users that are assigned to the case will be displayed to select from.

 

2.        The second will be all users that are both a Case Manager and Team Member role type.  This group of users are usually performing more administrative type activities in the application that may have access to all case and all teams.

 

-Select a user in the list

 

-Optionally add comments in the comment box

 

-Select the Assign button

 

Things to note:

 

-If a Task is not yet assigned, then the Assign screen will be presented when the user select the Assign/Transition Task node

 

 -To know if a user is not assigned to any Tasks, the Task header will display no user name on the right:
 

         

 

-If a Task is already assigned, then the Transition screen will be presented when the user selects the Assign/Transition Task node

 

 -To know if a user is assigned to any Tasks, the Task headers will display the user's name on the right:

 

 

 

-A user may bulk Assign/Transition Tasks if the Tasks are a part of the same Team Service.
