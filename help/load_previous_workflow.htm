Load Previous Workflow

 Top  Previous  Next

Instead of building custom project workflows as described in the Submit New Case Project section, the user can select the Load Prev Workflow button prior to adding custom tasks to the project workflow area under the New Project process:

The Load Prev Workflow button process allows the user to select a prior submitted project and carry over the workflow and associated instructions into a new project process.  This helps the user drastically reduce the time to submit a new project that shares the same workflow and instructions of a prior project.  All instruction from the previous project will be carried over unless the administration of IS-A-TASK some form elements to not carry over specific information.  The Admin can specify which form element data will not be carried over when a user using the Load Prev Workflow process by adjusting the Case Manager and Operation Teams form template process by editing the form element property and selecting the No Load Prev option.

-Select Load Prev Workflow button as soon as you enter the build workflow area

-Select a previous project in the list

-Select Load button, and the user will see the Project Workflow window auto-populate with tasks

Note: By default the grid will display a list of projects for the case the user is on.  Alternatively, the user can select another client/case they have access to in the dropdown list to display projects submitted under the select case.  This allows the user to copy a previous workflow and all instructions that were previously entered in another case over to the current case.

-Select Next button, and the user will see all of the prior project's workflow instructions already prepopulated

-Select Save button to save the project prior to Submitting the project.  This is a good idea if the user cannot Submit the project because the user needed to obtain additional instructions from the client.  The user can select the Saved Projects button in Case Projects area to relaunch the project workflow to finalize the instructions to submit.

-Select Submit button to finalize the submission process and the project is now live in the system and will appear in the Case Project Grid display
