Project Estimated Time

 Top  Previous  Next

The Project Estimated Time feature allows the user to estimate how long a task may take prior to submitting a new project, and these times are measured against the Projected Project Time and Actual Time the users are spending on completing Tasks.  This is an optional feature.

 

-By default, the Projected Project Time is always estimated as users enter the new project workflow process screen.  This is calculated based on the Due Date and Due Time the user enters on the previous screen of new project process:

 

 

-By default, as the users work through completing project tasks, the system will compare the Actual time the users take to complete tasks in a project against the Projected Project Time and display a project progress reports to the user.

 

 

-Optionally, the user may add Estimated Time values at the time of creating the project to add a third layer of reporting to the project progress report:

 

 

The process of adding Estimated Time values to a project workflow is the following:

 

 

-Select a Task in the Project Workflow section - this will add the entire time you enter in the Est Time window to the selected task

 

-Select one or more Tasks in the Project Workflow section - this will split the time you enter in the Est Time window between the selected tasks, and if there are media and media contents added to the task, the time will be according to the amount of media contents are added to a specific task.

 

-Select the icon for the Est Time window to display

 

 

-Enter the appropriate time value

 

-Select Update button

 

The Estimated Time value will now be displayed in the Project Workflow section:

 

 

-A new Estimated Project Time value will now be displayed next to the Projected Project Time

 

-If the Estimate Project Time exceeds the Projected Project Time, a Project Over Time value will be displayed and the user can either revise the Due Date and Time, or adjust the Est Time task values prior to submitting the project.

 

Note:  If the user selects the Load Prev Workflow option when building a project workflow, then any prior Estimated Time Values added to the prior project will carry over to new project process.
